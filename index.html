Features

Built for developers who value knowledge

Topic-Based Organization

Ask Claude to save to any topic — it creates folders and files automatically following your AGENT.md structure.

Chronological Tracking

Date-based filenames make it easy to track when you learned something. Watch your knowledge grow over time.

Pure Markdown

Readable, version-controllable, and editor-friendly. Your knowledge base works with any tool that understands Markdown.

Context7 Integration

Built-in Context7 support ensures you always get up-to-date library docs. Claude fetches current APIs and best practices automatically.

Git Native

Built for version control from day one. Track changes, branch experiments, and sync across machines effortlessly.

Zero Dependencies

No build process, no framework lock-in. Just files and folders. Works with any editor, any time, anywhere.

Structure

Simple by design

A filesystem structure that mirrors how you think. No databases, no hidden metadata—just human-readable files.

  • Topic folders in kebab-case (e.g., github-private-repos)
  • Date-prefixed files: YYYY-MM-DD-{description}.md
  • Frontmatter with title and date for consistency
  • Pure Markdown — readable anywhere
# Folder structure pattern

claude-chat-knowledge-base/
AGENT.md # Your custom Claude instructions
README.md # Project documentation
{topic-name}/ # Topic folder (kebab-case)
YYYY-MM-DD-{description}.md

# Real example from this repo:
sparkboard-chat/
2025-12-27-consolidated-docs.md

Quick Start

1

Clone the repository

Get started by forking or cloning the template.

$ git clone https://github.com/aungmyokyaw/claude-chat-knowledge-base.git
2

Customize AGENT.md (optional)

Add your personal Claude guidelines for consistent responses.

$ cat AGENT.md # Edit with your preferences
3

Start saving conversations

Chat with Claude CLI. When you want to save, just ask — Claude handles the rest.

$ claude
⟩ Help me build a React component...

⟩ Save this conversation to my knowledge base

# Claude creates the folder and saves the file
# following the structure in AGENT.md
4

Saved files include frontmatter

Each saved file starts with title and date metadata for consistency.

# {Title}

 **Date:** YYYY-MM-DD
**Topic:** {Topic name}

 {Content goes here...}
